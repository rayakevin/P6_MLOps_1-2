Préparation modeling dans les notebooks 02/03

  • prepare_modeling_tables() remplace les infinis par NaN.
  • L'identifiant SK_ID_CURR et la cible TARGET sont retirés des features.
  • Les colonnes à plus de 95 % de manquants sont supprimées sur la base du train.
  • Les colonnes constantes sont supprimées.
  • Train et test sont réalignés strictement colonne par colonne.
  • Les booléens texte true/false sont convertis en 0/1.
  • Les colonnes object résiduelles sont retirées par sécurité.
  • Les noms de colonnes sont normalisés pour éviter les caractères problématiques.

Nuance importante sur l'imputation

  • Le notebook 01 produit une preview model-ready via un SimpleImputer(strategy='median') afin de documenter une matrice finale exploitable sans manquants.
  • Dans les notebooks 02 et 03, on conserve les NaN quand les modèles ou pipelines savent les gérer.
  • Les modèles linéaires type SGDClassifier sont placés derrière un pipeline avec imputation médiane.
  • Les modèles d'arbres / boosting comme HistGradientBoosting et LightGBM tolèrent les manquants.

5. Notebook 02 - benchmark baseline et choix du candidat

Le notebook 02 reprend les fichiers application_train_full.parquet et application_test_full.parquet, effectue un split train / holdout stratifié, puis compare un petit ensemble de baselines lisibles.

Baselines comparées

  • DummyClassifier : baseline naïve, pour montrer qu'une forte accuracy n'a aucune valeur si le rappel sur la classe positive est nul.
  • SGDClassifier (log_loss) : baseline linéaire scalable, utile pour disposer d'une référence rapide et interprétable.
  • HistGradientBoosting : booster natif scikit-learn, compétitif en tabulaire et simple à expliquer.
  • LightGBM : baseline bonus plus puissante, candidate naturelle pour la suite.

Choix techniques importants.
  • Le coût métier est défini avec FN = 10 et FP = 1.
  • Le scoring de GridSearchCV n'est pas une simple AUC : chaque fold cherche d'abord le meilleur seuil sur la grille 0.10 -> 0.90.
  • Le notebook 03 tourne en séquentiel pour lightgbm_bonus afin d'éviter les TerminatedWorkerError liés à la duplication mémoire sous Windows.
  • Après audit de performance, le GPU est ciblé explicitement via la bonne plateforme OpenCL NVIDIA, et non l'iGPU Intel détecté par défaut.
Dans les artefacts actuellement présents, le meilleur seuil métier sur validation est 0.49. Sur le holdout, le coût métier reste très proche entre la baseline (0.4882), le modèle tuné à 0.5 (0.4888) et la version seuil métier (0.4889). Cela montre un gain technique modeste et un arbitrage métier fin plutôt qu’un saut de performance spectaculaire.
