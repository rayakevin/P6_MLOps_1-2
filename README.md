Convention de nommage des commit : 
`type: description courte`

Types à utiliser :

`fix`: corriger un beug,
`refactor`: simplifier la logique, 
`test`: ajouter des tests,
`docs`: documenter l'installation,
`chore`: mettre a jour les dependances.
